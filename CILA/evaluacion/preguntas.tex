\section*{Módulo I -- Entorno GNU/Linux}

\subsection*{Historia}

\begin{itemize}

\item ¿Quién, cuándo, dónde, cómo y por qué creo Linux?

\item ¿Quién, cuándo, dónde, cómo y por qué creo el movimiento GNU?

\item  ¿Qué diferencia  hay entre  los términos  ``Linux'', ``GNU''  y
``GNU/Linux''?

\item Lista al menos 10 distribuciones de GNU/Linux, explicando en qué
consiste  cada una.  Subirá  nota buscar  distribuciones inusuales  en
lugar de listar las distribuciones más conocidas .

\end{itemize}

\subsection*{Intérprete de comandos}

Cuando hayas terminado con los demás ejercicios (de todos los módulos)
y  tengas  preparados  todos  los  ficheros  para  enviarlos,  haz  lo
siguiente:

\begin{enumerate}

\item Crea un nuevo directorio con  tu nombre y apellidos sin utilizar
tildes y substituyendo los espacios por subrayados. Por ejemplo, si te
llamas ``Iván García'' la carpeta debe llamarse {\tt Ivan\_Garcia}

\item Entra en  este directorio y crea un directorio  para cada módulo
en que  te hayas matriculado:  {\tt Modulo\_1}, {\tt  Modulo\_2}, {\tt
Modulo\_3}, etc.

\item  En   cada  uno   de  estos   directorios  copia   los  ficheros
correspondientes a las preguntas de cada módulo.

\item  Comprime  el primer  directorio  (con  tu nombre  y  apellidos)
mediante el comando {\tt tar}. El uso básico de este comando es:

\begin{verbatim}
$ tar cfz Nombre_Apellidos.tar.gz Nombre_Apellidos/
\end{verbatim}

Subirá nota si en lugar de comprimir  el paquete tar con {\tt gzip} lo
comprimes con  {\tt bzip2}, consultando  cómo hacerlo en la  página de
manual de {\tt tar}. Esto lo miras con el comando {\tt man}

\end{enumerate}

\subsection*{Internet}

Navegar   con    un   navegador    (galeon,   mozilla,    netscape   o
konqueror)  y   crear  una  colección   de  al  menos   15  marcadores
(bookmarks)  organizados  en  carpetas.  Para  darnos  tus  marcadores
envíanos    el    fichero   {\tt    .netscape/bookmarks.html},    {\tt
.mozilla/*/*.slt/bookmarks.html}, {\tt  .galeon/bookmarks.xbel} o {\tt
.kde/share/apps/konqueror/bookmarks.xml}  según utilices  el navegador
netscape, mozilla, galeon o konqueror respectivamente.

\section*{Módulo III}

\subsection*{Edición de gráficos}

Elije al menos una de las siguientes opciones:

\begin{itemize}

\item Hacer un diagrama con DIA, por  ejemplo un esquema de una red de
ordenadores, un circuito eléctrico, un  diagrama de flujo o un esquema
en UML. El fichero debe estar en formato de DIA (.dia)

\item Dibujar un plano de una habitación mueblada con QCad, utilizando
una  capa para  las paredes,  otra para  los muebles  y señalando  las
dimensiones de los objetos en una capa de acotaciones. El fichero debe
estar en formato DXF.

\item Crear una  imagen o retocar una fotografía  con GIMP, utilizando
un más de  una capa y algún  filtro. El fichero debe  estar en formato
XCF (nativo de GIMP), así que no debe ser muy grande.

\end{itemize}

Subirá  nota  hacer  más  de  una  de  estas  opciones,  o  hacerla(s)
utilizando técnicas gráficas que no hayas aprendido en el curso.

\subsection*{HTML}

Escribe un  fichero HTML que contenga  un resumen de los  ficheros que
nos entregas. En el fichero HTML el nombre de cada fichero debe ser un
enlace  a  donde  se  encuentre  este fichero.  Por  ejemplo,  si  los
directorios de los  módulos se llaman {\tt mod1}, {\tt  mod2}, etc. en
el fichero el enlace debe ser algo como
\verb|<a href="../mod4/fichero.txt">fichero.txt</a>|

valorará que tengan un contenido interesante y/o un diseño cuidado.

\subsection*{R}

Genera una muestra de 50 números aleatorios que sigan una distribución
normal  con   media  y   varianza  aleatorias.  Haz   una  descripción
estadística  de la  muestra  incluyendo  la media,  la  mediana y  los
cuartiles. Representa la muestra gráficamente mediante un histograma.

Si quieres subir nota, haz un contraste de hipótesis a la muestra para
contrastar si la media aritmética muestral es una estimación aceptable
de  la media  con  la  que generaste  la  muestra.  Recuerda que  este
contraste requiere un test de  normalidad. Ayúdate de los gráficos Q-Q
(cuartil-cuartil)  y  represéntala  en  un diagrama  de  sectores  por
intervalos utilizando los cuartiles como extremos de los intervalos.

Debes  introducir  los comandos  necesarios  para  esto en  un  script
(fichero de texto {\tt script.R}) y adjuntárnoslos. Los gráficos no es
necesario que  nos los  adjuntes, ya  que deben  ser generados  por el
script.

\subsection*{Compiladores y Makefile}

Escribe un  {\tt Makefile}  que compile  al menos  un ejemplo  en cada
lenguaje  (Pascal, Fortran,  C, C++)  de  los ejemplos  de código  que
incluye el  libro. Recuerda  que estos  ejemplos puedes  obtenerlos en
{\tt  http://cila.gulic.org/ejemplos.php}  comprimidos en  un  fichero
{\tt ejemplos.tar.gz} que puedes descomprimir con el comando

\begin{verbatim}
$ tar xfz ejemplos.tar.gz
\end{verbatim}

Añadir un  objetivo {\tt  clean} que borre  los ejecutables  y objetos
resultantes de la compilación.

Para subir nota, separar las  compilaciones de los distintos programas
(ejemplos de código) para que se puedan compilar por separado. Esto se
hace definiendo un objetivo para  cada ejemplo, y haciendo depender el
objetivo {\tt all} de todos ellos.

\subsection*{Depuradores}

\subsection*{Bash}

Escribe  un script  en Bash  que  muestre información  sobre un  usuario
especificado. El script debe:

\begin{itemize}

\item Mostrar un mensaje que solicite el nombre de un usuario.

\item Leer el nombre del usuario desde el teclado.

\item Mostrar los resultados del  comando {\tt who} (listado de usuarios
activos) referentes al usuario indicado.  Para ello basta con entubar la
salida de un comando {\tt who} sencillo a través del comando {\tt grep}.
(Ej: {\tt who | grep $<$usuario$>$})

\item Mostrar los resultado del comando {\tt ps} (listado de procesos en
ejecución) referentes al  usuario indicado. Para ello  basta con entubar
la salida de un comando {\tt ps -aux} sencillo a través del comando {\tt
grep}. (Ej: {\tt ps -aux | grep $<$usuario$>$})

\end{itemize}

Si quieres subir  nota escribe un script que  muestre información acerca
de cada uno de los directorios especificados en la línea de comandos. El
script debe:

\begin{itemize}

\item Mostrar el número de archivos  y subdirectorios de cada uno de los
directorios.

\item Listar todos los archivos de  cada directorio que tengan un tamaño
de cero bytes (usar una opción del comando {\tt find} para hacerlo).

\item  Usar el  comando {\tt  du} para  mostrar la  cantidad de  espacio
utilizado por cada directorio.

\begin{itemize}
